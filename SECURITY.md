# Security Policy

## Supported Versions

Only the latest release is supported.

## Reporting A Vulnerability

Open a private security advisory on GitHub or contact the repository owner directly.

Do not publish exploit details publicly until the issue has been reviewed.

## Operational Notes

This project downloads and executes a release binary. Use HTTPS release URLs, keep the GitHub account protected with two-factor authentication, and publish release notes for every EXE update.

External benchmark tools may require their own license agreements. Configure only trusted vendor URLs in `config.py`.
